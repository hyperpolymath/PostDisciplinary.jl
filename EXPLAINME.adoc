== Claim 1: Universal Ontology Linking Entities Across Disciplines

From README:
____
Universal Ontology: A common data schema for linking "Entities" (People, Events, Proofs, Claims) across all Hyperpolymath libraries.
____

**How it's proven:**
- **Core struct**: `LinkedEntity` defined at line 78-88 in `src/PostDisciplinary.jl` with fields: `id::UUID`, `source_library::Symbol`, `original_id::Symbol`, `kind::Symbol`, `metadata::Dict{Symbol, Any}`
- **Graph container**: `ResearchProject` (lines 94-108) wraps a `MetaGraph` where vertices are `LinkedEntity` objects with cross-disciplinary relationships
- **Linking operations**: `add_link!(project, entity1, entity2, rel)` at line 126 establishes edges between entities from different libraries with relationship metadata
- **Synthesis**: `generate_synthesis(project)` at line 137 extracts all entities, groups by `source_library` and `kind`, computes graph density
- **Key path**: All modules (`memetics.jl`, `meta_analysis.jl`, `methodology.jl`) inherit from this ontology via UUID-keyed entity IDs

**Caveat:** The `@link` macro (lines 115-120) is syntactic sugar only—it echoes progress messages but does not yet validate entity type compatibility or enforce cross-library interface contracts.

== Claim 2: Meta-Analysis with Effect Size Aggregation and Heterogeneity Testing

From README:
____
Integrated Research Graphs: Link evidence from InvestigativeJournalist.jl with models from Cliodynamics.jl. Cross-Theory Verification: Verify a social value from Axiology.jl using a formal proof in Axiom.jl.
____

**How it's proven:**
- **Meta-analysis module**: `src/meta_analysis.jl` implements effect size aggregation and heterogeneity (Q-statistic) testing
- **Exports**: `EffectSize`, `aggregate_effects`, `heterogeneity_q` at line 63
- **Integration path**: `MetaAnalysis` submodule is imported (line 47) and used by orchestration workflows
- **Storage integration**: `VeriSimBridge` (line 36) and `LithoglyphBridge` (line 37) provide persistence bridges to VeriSimDB and Lithoglyph storage systems

**Caveat:** Actual implementations of aggregation algorithms and Q-statistic calculation are currently minimal stubs. The framework for connecting to external evidence repositories (InvestigativeJournalist.jl, Cliodynamics.jl) is defined but integration is incomplete.

== Dogfooded Across The Account

PostDisciplinary.jl is the **orchestration hub** for the ecosystem:
- **Cliodynamics.jl**: Provides historical dynamics data (`LinkedEntity` with `kind=:historical_event`)
- **Causals.jl**: Provides causal model data (`LinkedEntity` with `kind=:causal_graph`)
- **Axiom.jl**: Provides formal proofs (`LinkedEntity` with `kind=:proof`)
- **InvestigativeJournalist.jl**: Evidence source (planned integration)
- **Axiology.jl**: Value systems (planned integration)

All integrations use the `LinkedEntity` ontology. Same orchestration pattern used in `hypatia` (CI/CD meta-layer).

== Critical Invariants (What Must Not Break)

1. Every entity in the graph must have a unique UUID and valid `source_library` symbol
2. The `LinkedEntity` ontology must remain stable—adding fields breaks backward compatibility
3. All cross-library integrations must use UUID references, not hard-coded object pointers
4. Synthesis reports must always be reproducible given the same input graph
